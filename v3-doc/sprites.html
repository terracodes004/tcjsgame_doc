Sprites in TCJSgame

Sprites are animated images used to represent characters, enemies, or visual effects in a game. In TCJSgame v3, sprites are managed through the Sprite class. This class takes a sprite sheet image and automatically cycles through frames to create smooth animations.

A sprite sheet is a single image containing multiple frames of animation, placed side by side. The Sprite class requires you to specify the frame width, frame height, number of frames, and the animation speed. Once set up, the sprite can be updated every frame and drawn on the canvas.

Example


// Load sprite image
let image = new Image();
image.src = "character.png";

// Create a new Sprite
let playerSprite = new Sprite(image, 64, 64, 4, 10);

// Update function
function update() {
    display.clear();

    // Update sprite animation
    playerSprite.update();

    // Draw sprite at (100, 100)
    playerSprite.draw(display.context, 100, 100);
}

In this example, the sprite sheet character.png has 4 frames, each 64Ã—64 pixels. The frameSpeed of 10 means the sprite will change frames every 10 updates, creating a controlled animation speed.

Note: Sprites in TCJSgame are independent from Component. They do not have built-in movement or physics. If you want your sprite to move, you should combine it with a Component or update its position manually in the update() loop. This separation gives you more flexibility to use sprites purely for visual effects or attach them to game objects.
